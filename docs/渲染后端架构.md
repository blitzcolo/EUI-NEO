# 渲染后端架构

本文定义 EUI-NEO 的最终渲染后端架构。后续改动必须按本文收束，目标是形成稳定、可维护的 OpenGL/Vulkan 双渲染后端，而不是继续堆临时胶水。

## 目标

EUI-NEO 要同时支持：

- GLFW / SDL2 双窗口后端。
- OpenGL / Vulkan 双渲染后端。
- 同一套 DSL、组件、layout、text/image CPU 逻辑。
- OpenGL 与 Vulkan 输出同一 DSL 时视觉语义一致。

核心原则：

- 窗口后端只管理窗口、输入、IME、剪贴板、光标、图标、native handle、父子窗口和 modal 行为。
- 渲染后端只管理 GPU API、surface/context、swapchain/framebuffer、frame lifecycle、render cache、primitive/text/image GPU 资源。
- Runtime 只管理 DSL 树、layout、事件、dirty rect、动画和 render dispatch，不知道 GLFW/SDL2/OpenGL/Vulkan 细节。
- Public facade、components、examples 不暴露 OpenGL/Vulkan/GLFW/SDL2 专属类型。
- 架构收束时同步删除旧路径、旧 include、旧开关、旧文档，不保留长期兼容壳。

## 方向依赖

```text
examples / user app
  |
include/eui_neo.h, include/eui/*
  |
core/app
AppRunner / MainWindowRuntime / DslWindowManager
  |
  +--> core/window  -----> GLFW / SDL2 / native window handles
  |
  +--> core/input   -----> backend-neutral input events
  |
  +--> core/dsl runtime
          |
          +--> core/render public API
                   |
                   +--> core/render/opengl  -----> OpenGL
                   |
                   +--> core/render/vulkan  -----> Vulkan
```

边界要求：

- `core/app` 可以调 `core/window`、`core/input`、`core/render` 公共接口，但不能 include `<glad/glad.h>` 或 Vulkan 头。
- `core/window` 可以 include GLFW/SDL2 头，但不能创建 shader、texture、swapchain 或 render cache。
- `core/render` 公共层只能放后端无关接口和类型。
- `core/render/opengl` 可以 include glad/OpenGL 头。
- `core/render/vulkan` 可以 include Vulkan 头。
- `include/`、`components/`、`examples/` 不得 include 后端专属头。

## 模块职责

### Public Facade

稳定入口：

- `include/eui_neo.h`
- `include/eui/*.h`

不得暴露：

- `GLFWwindow*`
- `SDL_Window*`
- `<glad/glad.h>`
- `<vulkan/vulkan.h>`
- OpenGL/Vulkan 具体后端类

### AppRunner / App Main

职责：

- 统一主循环状态机。
- 统一主窗口和子窗口 update/render 流程。
- 处理帧时间、FPS 标题、按需渲染、主动节流。
- 调用窗口事件、runtime update、render backend frame。

非职责：

- 不创建 OpenGL context。
- 不创建 Vulkan instance/surface/swapchain。
- 不直接调用 `gl*` / `vk*`。
- 不实现 primitive/text/image 绘制。

### WindowBackend

职责：

- 创建、销毁、显示、隐藏、raise、focus 窗口。
- 管理 GLFW/SDL2 的窗口 flag 和 GL/Vulkan window hint。
- 查询 window size、framebuffer size、DPI、pointer scale。
- 翻译窗口、鼠标、键盘、滚轮、文本输入事件。
- 输入法候选框定位、剪贴板、光标、窗口图标。
- 提供 `NativeWindowInfo` 给平台能力和 Vulkan surface 创建。
- 处理父子窗口、modal、置顶/focus 等窗口语义。

非职责：

- 不绘制。
- 不持有 GPU resource。
- 不持有 render cache。
- 不在事件泵里写 OpenGL/Vulkan draw/present 分支。

### Runtime

职责：

- DSL tree compose/layout。
- 事件派发、focus、IME rect、dirty rect、动画状态。
- 调用 primitive/text/image facade。
- 通过 `RenderBackend` 下发后端无关 draw command。

非职责：

- 不创建 OpenGL/Vulkan resource。
- 不包含 GLFW/SDL2 类型。
- 不包含 OpenGL/Vulkan 分支。

### RenderBackend

职责：

- 管理 OpenGL context 或 Vulkan instance/surface/device/swapchain。
- 管理 frame lifecycle：begin frame、clear、render cache、draw、present、resize、cleanup。
- 管理后端专属 primitive/text/image GPU 资源。
- 处理 minimized/zero-size 窗口。

非职责：

- 不读取窗口事件。
- 不决定窗口关闭、隐藏托盘、modal focus。
- 不修改 DSL tree 或 layout。

## 最终目录结构

短期可以逐步迁移，但最终结构以这里为准。

```text
core/app/
  app_runner.h
  main_window_runtime.h
  dsl_window_manager.h
  dsl_window_runtime.h
  glfw_app_main.cpp
  sdl2_app_main.cpp

core/input/
  input_types.h
  input_state.h

core/window/
  window_types.h
  window_backend.h/.cpp

core/platform/
  async.h/.cpp
  network.h/.cpp
  platform.h/.cpp
  native_bridge.*
  tray_bridge.*
  ime_bridge.*

core/render/
  render_backend.h/.cpp
  render_surface.h
  render_types.h
  primitive.h/.cpp
  primitive_geometry.h
  text.h/.cpp
  text_types.h
  image.h
  image_types.h
  image.cpp
  image_source.h/.cpp
  stb_image_impl.cpp

core/render/opengl/
  opengl_backend.h/.cpp
  opengl_primitives.cpp
  opengl_text.cpp
  opengl_image.cpp

core/render/vulkan/
  vulkan_backend.h/.cpp
  vulkan_cache.cpp
  vulkan_primitives.cpp
  vulkan_text.cpp
  vulkan_image.cpp
  vulkan_*_shaders.h
  shaders/
```

文件职责：

- `render_backend.h/.cpp`：后端无关接口、active backend、backend factory、当前窗口需要的 `RenderApi`。
- `render_surface.h`：窗口到渲染后端的握手数据。
- `primitive.cpp`：后端无关 primitive 状态、transform、dispatch。
- `primitive_geometry.h`：圆角矩形、shadow、polygon 等 CPU 几何语义。
- `text.cpp`：字体解析、fallback、HarfBuzz shaping、metrics、glyph vertices、atlas CPU 数据。
- `image.cpp`：`ImagePrimitive` 状态、fit/cover、transform、GIF 帧推进、静态图 texture cache 和后端无关 image draw dispatch。
- `image_source.*`：图片路径解析、remote/Bing 下载、SVG 栅格化、静态图/GIF CPU 解码缓存。
- `opengl_primitives.cpp` / `vulkan_primitives.cpp`：各自的 primitive/backdrop GPU pipeline、vertex buffer、draw。
- `opengl_text.cpp` / `vulkan_text.cpp`：各自的 text atlas upload、descriptor/pipeline、vertex buffer、draw。
- `opengl_image.cpp` / `vulkan_image.cpp`：各自的 image texture upload、descriptor/pipeline、vertex buffer、draw。
- `vulkan_cache.cpp`：Vulkan render cache image/framebuffer、copy/采样 resolve pipeline、cache 资源释放。
- `opengl_backend.cpp` / `vulkan_backend.cpp`：各自的 context/surface/swapchain/frame lifecycle 总控。

## 后端选择

CMake 只通过两个维度选择后端：

- `EUI_WINDOW_BACKEND=glfw|sdl2`
- `EUI_RENDER_BACKEND=auto|opengl|vulkan`

策略：

- `auto`：默认。Vulkan SDK/loader 可用时选择 Vulkan，否则回退 OpenGL。
- `opengl`：强制 OpenGL。
- `vulkan`：强制 Vulkan。Vulkan SDK/loader 或 EUI Vulkan 后端不可用时配置失败，不静默回退。

## 当前收束任务

按下面顺序改，避免继续把文件揉成一团。

1. 收紧构建开关

- 已完成：移除 Vulkan experimental gate。
- 已完成：Vulkan presets 只设置 `EUI_RENDER_BACKEND=vulkan`。
- 已完成：`auto` 保持安全回退，强制 `vulkan` 保持明确失败。

2. 拆分 Vulkan text

- 已完成：`vulkan_text.cpp` 保留并补实，不能删除。
- 已完成：从 `vulkan_backend.cpp` 移出：
  - `drawText`
  - `ensureTextPipeline`
  - `textAtlasNeedsUpload`
  - `ensureTextAtlas`
  - `ensureTextDescriptor`
  - `ensureTextVertexBuffer`
  - `destroyTextPipeline`
  - `destroyTextResources`
- 拆分只移动实现位置，不改变渲染行为。

3. 拆分 image

- 已完成：`ImagePrimitive` CPU/GPU 桥接逻辑已收束到共享 `core/render/image.cpp`。
- 已完成：`opengl_image.cpp` / `vulkan_image.cpp` 只保留各自 image texture upload、descriptor/pipeline、vertex buffer 和 draw。
- 已完成：从 `vulkan_backend.cpp` 移出：
  - `createTexture`
  - `updateTexture`
  - `destroyTexture`
  - `drawTexture`
  - `ensureImagePipeline`
  - `ensureImageDescriptor`
  - `ensureImageVertexBuffer`
  - `destroyImagePipeline`
  - `destroyImageResources`
- OpenGL/Vulkan image 的路径解析、remote/Bing、SVG、静态图/GIF CPU 解码继续共用 `image_source.*`。

4. 收紧 Vulkan backend 主文件

`vulkan_backend.cpp` 最终只保留：

- instance/device/surface/swapchain。
- render pass/framebuffer/command buffer。
- frame acquire/submit/present。
- resize / cleanup 总控。
- shared utility 只保留当前文件内确实需要的部分。

- 已完成：primitive/backdrop pipeline 已从 `vulkan_backend.cpp` 拆到 `vulkan_primitives.cpp`。
- 已完成：render cache image/framebuffer 和 resolve pipeline 已从 `vulkan_backend.cpp` 拆到 `vulkan_cache.cpp`。

5. 清理残留

- 删除没有职责的空翻译单元。
- 删除旧 include、旧 CMake source、旧文档引用。
- 不删除未来分区文件，例如 `vulkan_text.cpp` 这类应该补实的文件。

## 禁止事项

- 禁止新增 `main_vulkan.cpp`、`main_sdl2_vulkan.cpp` 这类并行 main。
- 禁止把 Vulkan 分支塞进 GLFW/SDL2 事件泵。
- 禁止用 bounding box、纯色块、no-op 冒充 Vulkan primitive/text/image 完成。
- 禁止 public include 透出 OpenGL/Vulkan/GLFW/SDL2 专属类型。
- 禁止只包一层 helper 就宣称架构收束。
- 禁止保留长期兼容壳和旧路径。
- 禁止未确认最终职责就删除文件。
